# Skills Reference

Skills are invoked by typing their name as a keyword in Copilot Chat.
They live in `~/.agents/skills/<name>/SKILL.md`.

| Skill | When to use |
|---|---|
| `audit` | Full quality report (accessibility, performance, responsive design) |
| `polish` | Final pre-ship quality pass — alignment, spacing, consistency |
| `arrange` | Fix layout, spacing, and visual hierarchy |
| `animate` | Add motion and micro-interactions |
| `colorize` | Add strategic color to monochromatic interfaces |
| `typeset` | Improve typography — font choices, hierarchy, sizing, weight |
| `distill` | Simplify and remove unnecessary complexity |
| `bolder` | Make designs more visually impactful |
| `quieter` | Tone down overly aggressive or heavy designs |
| `harden` | Error handling, edge cases, i18n, text overflow robustness |
| `optimize` | Performance — loading speed, rendering, bundle size |
| `frontend-design` | Build polished new UI components or pages from scratch |
| `normalize` | Align to the design system and ensure consistency |
| `extract` | Pull out reusable components, design tokens, and patterns |
| `onboard` | Empty states, first-run UX, onboarding flows |
| `clarify` | Improve UX copy, error messages, labels, and microcopy |
| `critique` | UX/design evaluation with actionable feedback |
| `adapt` | Make interfaces responsive and cross-platform |
| `delight` | Add personality and memorable moments |
| `overdrive` | Technically ambitious implementations (shaders, physics, scroll-driven effects) |
| `find-skills` | Discover and install new skills |
| `teach-impeccable` | One-time setup to establish persistent design guidelines |
